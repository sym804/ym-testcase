# 이슈 관리 규칙

## 이슈 ID 체계

| 접두사 | 분류 | 예시 |
|---|---|---|
| `BUG-` | 기능 버그 | BUG-001 |
| `SEC-` | 보안 이슈 | SEC-001 |
| `ENC-` | 인코딩/한글 깨짐 | ENC-001 |
| `PERF-` | 성능 이슈 | PERF-001 |
| `ENH-` | 개선 요청 | ENH-001 |

## 영역 분류

| 영역 | 대상 | GitHub 라벨 |
|---|---|---|
| **Frontend** | TSX 컴포넌트, UI/UX, 빌드, E2E, 인코딩 | `frontend` |
| **Backend** | API, 비즈니스 로직, 인증/권한, pytest | `backend` |
| **DB** | 테이블/컬럼, 마이그레이션, 쿼리 최적화, deleted_at | `db` |
| **보안** | 인증 우회, 크레덴셜 노출, CORS, CSRF, Rate Limiting | `security` |
| **기타** | 문서(README/라이선스), 설정, 스크립트, 환경 구성 | `etc` |

## 심각도 기준

| 심각도 | 기준 | 예시 |
|---|---|---|
| **Block** | 서비스 전체 장애, 작업 진행 불가. workaround 없음 | 서버 미기동, 로그인 불가, DB 연결 실패 |
| **Critical** | 핵심 기능 마비, 데이터 유실/손상. 다른 작업은 가능 | 빌드 실패, 인증 우회, TC 저장 안 됨, 특정 페이지 진입 불가 |
| **Major** | 주요 기능 오류, 중요 UI 문제 | 대시보드 통계 오류, Import 실패, 권한 불일치, deleted_at 필터 누락 |
| **Minor** | 사소한 기능 오류, 경미한 UI 문제 | 토스트 중복, 날짜 포맷 오류, 정렬 안 됨, 에러 메시지 영문 표시 |
| **Trivial** | 기능 무관한 외관/텍스트, 개선 사항 | 오타, 색상 미세 조정, 코드 정리, README 문구 수정 |

### 심각도 판단 기준

- **Block vs Critical**: 로그인이 안 되면 Block(아무것도 못 함), 로그인은 되는데 TC 관리 페이지 진입이 안 되면 Critical(다른 페이지는 사용 가능)
- **Critical vs Major**: 기능이 완전히 안 되면 Critical, 기능은 되는데 결과가 틀리면 Major
- **Major vs Minor**: 업무에 지장이 있으면 Major, 불편하지만 우회 가능하면 Minor
- **Minor vs Trivial**: 기능 관련이면 Minor, 기능과 무관한 외관/텍스트이면 Trivial

### 분류별 심각도 범위

| 분류 | 사용 가능 심각도 | 비고 |
|---|---|---|
| BUG, ENC, PERF | Block, Critical, Major, Minor, Trivial | 5단계 전체 |
| SEC (보안 이슈) | Block, Critical, Major, Minor | 최소 Minor 이상 (Trivial 불가 — 보안은 사소하지 않음) |
| DB 관련 이슈 | Block, Critical, Major | 최소 Major 이상 (DB 문제는 데이터 정합성 직결) |
| ENH (개선 요청) | Major, Minor, Trivial | 3단계만 (Block/Critical 불가 — 개선은 장애/마비가 아님) |

## 상태 관리

| 상태 | 의미 | 전이 조건 |
|---|---|---|
| **Open** | 신규 등록, 작업 필요 | 이슈 발견 시 |
| **In Progress** | 개발 수정 중 | 담당자 지정 후 작업 시작 |
| **Ready for QA** | 수정 완료, QA 검증 대기 | 개발자가 수정 완료 후 |
| **Verified** | QA 검증 통과 | QA가 수정 확인 후 |
| **Released** | 운영 배포 완료 | 릴리즈에 포함되어 배포 후 |
| **Reopened** | 검증 실패로 재오픈 | QA 검증 중 재현되거나 미수정 확인 시 |
| **Deferred** | 장기 과제로 보류 | 현재 릴리즈 범위 외, 사유 기록 |
| **Won't Fix** | 수정하지 않음 | 설계 의도, 재현 불가 등 사유 기록 |

### 상태 흐름

```
Open → In Progress → Ready for QA → Verified → Released
                         ↓                ↓
                      Reopened ←──────────┘
                         ↓
                    In Progress (재작업)

Open → Deferred (보류)
Open → Won't Fix (수정 안 함)
```

## 필수 기록 항목

- Issue ID, 제목, 영역(Frontend/Backend/DB/보안/기타), 심각도, 상태
- 발견일, 발생 버전 (컴포넌트 버전)
- 재현 방법 / 개선 사항 (필수)
- 원인 및 수정 / 개선 사유 (필수)
- 영향 범위 (선택)
- 수정 버전 (컴포넌트 버전, 수정 완료 시 필수)

## 버그 작성 양식

분류가 `BUG-`인 경우 아래 양식을 따른다.

| 항목 | 필수 여부 | 설명 |
|---|---|---|
| **발생 버전** | 필수 | 버그가 발생한 버전 (예: v0.7.1.0) |
| **사전 조건** | 선택 | 있으면 기술, 없으면 "없음"으로 기록 |
| **재현 방법** | 필수 | 재현 스텝을 번호로 기술 |
| **발생 원인 및 수정사항** | 필수 | 원인 분석과 수정 내용 |
| **영향 범위** | 선택 | 수정으로 인한 영향 범위 기술 |
| **수정 버전** | 필수 | 수정이 반영된 버전 |

### 버그 작성 예시

```
[BUG-095] 대시보드 차트 색상 다크모드에서 안 보임

- 발생 버전: v0.7.0.0
- 사전 조건: 다크모드 활성화 상태
- 재현 방법:
  1. 프로젝트 상세 → 대시보드 탭 이동
  2. 결과 분포 차트 확인
  3. 차트 배경색과 데이터 색상이 동일하여 구분 불가
- 발생 원인 및 수정사항: CSS 변수가 라이트모드 값으로 고정되어 있었음 → hex 값으로 라이트/다크 분리 적용
- 영향 범위: 대시보드 차트 전체 (결과 분포, 라운드별 비교)
- 수정 버전: v0.7.1.0
```

## 이슈 검증 원칙

- 이슈 상태는 **코드 기반 검증** 후 변경 (리뷰 문서만으로 판단 금지)
- Ready for QA → Verified 전환은 **QA 담당자**만 수행
- Released 전환은 **릴리즈 노트에 포함된 후** 수행
- Reopened 시 재현 절차와 환경 정보 기록 필수
- 정기적으로 Open/In Progress 이슈 실제 상태 재검증

## 개선(ENH) 작성 양식

분류가 `ENH-`인 경우 아래 양식을 따른다.

| 항목 | 필수 여부 | 설명 |
|---|---|---|
| **발생 버전** | 필수 | 개선 대상 버전 (예: v0.7.1.0) |
| **개선 사항** | 필수 | 개선 내용 기술 |
| **개선 사유** | 필수 | 왜 개선이 필요한지 사유 |
| **영향 범위** | 선택 | 개선으로 인한 영향 범위 기술 |
| **수정 버전** | 필수 | 개선이 반영된 버전 |

### 개선 작성 예시

```
[ENH-012] Import 버튼 라벨 변경 및 Markdown 파일 지원

- 발생 버전: v0.7.1.0
- 개선 사항: Import 버튼 라벨을 "Excel Import"에서 "Import"로 변경, .md 파일 임포트 기능 추가
- 개선 사유: Markdown 기반 TC 작성이 증가하는 추세에 맞춰 .md 파일 직접 임포트 지원 필요
- 영향 범위: TC 관리 툴바, Import 플로우, 사용자 매뉴얼
- 수정 버전: v0.8.0.0
```
