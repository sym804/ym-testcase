# TC Manager 프로젝트 규칙

## 프로젝트 규칙 (반드시 따를 것)

아래 `rules/` 폴더의 규칙 문서를 반드시 참고하고 따라야 합니다:

- `rules/versioning.md` — 버전 체계 (`system.feature.fix.patch`, 컴포넌트별 독립 버전)
- `rules/release_process.md` — 릴리즈 프로세스 (체크리스트, 롤백 절차)
- `rules/tc_writing_guide.md` — TC 작성 가이드 (ID 규칙, 우선순위, FAIL 판정)
- `rules/issue_management.md` — 이슈 관리 (ID 체계, 심각도, 상태 흐름)
- `rules/project_naming.md` — 프로젝트/테스트 런 네이밍 규칙

## 기능 추가 시 필수 체크리스트

기능을 구현하면 반드시 아래를 함께 수행:
1. 코드 구현 + TypeScript 타입 체크 통과
2. 테스트 작성 및 수행 (API + E2E)
3. 사용자 매뉴얼 업데이트 (`UserManualPage.tsx`) + 스크린샷
4. 회귀 테스트 체크리스트 엑셀에 TC 추가
5. 테스트 데이터/비밀번호 변경 시 반드시 원복

## 서버 포트

- 백엔드: **8008** (8000 아님 — 다른 프로젝트 전용)
- 프론트엔드: **5173**

## 버튼 배치 규칙

- 액션 버튼(추가/삭제 등)은 관련 테이블 바로 위에 배치
- 페이지 최상단이나 다른 섹션에 두지 말 것

## 테스트 규칙

- 테스트 중 비밀번호, DB 데이터 변경 시 테스트 완료 후 반드시 원복
- rate limit 주의: 로그인 실패 10회 시 5분 잠금 (서버 재시작으로 해제)
