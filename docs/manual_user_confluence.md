# TC Manager 사용자 매뉴얼

> **최종 업데이트**: 2026-03-18
> **버전**: 0.4.0.0
> **대상**: 전체 사용자

---

## 1. 개요

**TC Manager**는 테스트 케이스(TC)의 관리, 테스트 수행, 결과 분석 및 리포트 생성을 위한 웹 기반 테스트 관리 시스템입니다.

**주요 기능:**
- 프로젝트 단위 테스트 케이스 관리 (CRUD, Excel Import/Export)
- 테스트 수행 및 결과 기록 (키보드 단축키, 첨부파일 지원)
- 테스트 런 간 비교 (변경/리그레션 감지)
- 대시보드 (도넛 차트, 바 차트, 히트맵 등 분석)
- 리포트 생성 (PDF / Excel 다운로드)
- 역할 기반 접근 제어 (Admin, QA Manager, User)
- 비밀번호 정책 관리 (최소 8자, 강제 변경)

## 2. 로그인 / 회원가입

### 2-1. 로그인

> 📎 `01_login_page.png`

1. 아이디(사용자명)를 입력합니다.
2. 비밀번호를 입력합니다.
3. **로그인** 버튼을 클릭합니다.
4. 성공 시 프로젝트 목록 페이지로 이동합니다.

💡 **TIP**: 로그인 실패 시 5분간 최대 10회까지 시도할 수 있습니다. 초과 시 일시적으로 잠깁니다.

### 2-2. 회원가입

> 📎 `02_register_page.png`

1. 로그인 화면 하단의 **"회원가입"** 링크를 클릭합니다.
2. 아이디, 표시명, 비밀번호를 입력합니다.
3. **회원가입** 버튼을 클릭합니다.

⚠️ **비밀번호 규칙**: 비밀번호는 **최소 8자** 이상이어야 합니다.

⚠️ **참고**: 최초 가입 사용자는 자동으로 **Admin** 역할이 부여됩니다. 이후 가입자는 **User** 역할로 시작하며, 관리자에게 역할 변경을 요청해야 합니다.

### 2-3. 비밀번호 변경

1. 로그인 후 사용자 정보 영역에서 **비밀번호 변경**을 선택합니다.
2. 현재 비밀번호를 입력합니다.
3. 새 비밀번호를 입력합니다 (최소 8자).
4. **변경** 버튼을 클릭합니다.

⚠️ **강제 변경**: 관리자가 비밀번호를 초기화한 경우, 다음 로그인 시 비밀번호 변경이 강제됩니다. 변경 완료 전까지 다른 기능을 사용할 수 없습니다.

💡 **TIP**: 새 비밀번호는 현재 비밀번호와 동일할 수 없습니다.

### 2-4. 사용자명 중복 확인

회원가입 시 아이디를 입력하면 실시간으로 사용 가능 여부가 표시됩니다. 이미 등록된 아이디는 사용할 수 없습니다.

## 7. 테스트 수행

> 📎 `09_testrun_tab.png`

### 7-1. 테스트 런 생성

1. 좌측 패널 하단의 **"+ New Test Run"** 버튼을 클릭합니다.
2. 런 이름, 라운드(R1/R2/...), 버전, 환경을 입력합니다.
3. 생성 시 프로젝트의 모든 TC가 **NS(미실행)** 상태로 포함됩니다.

### 7-2. 테스트 런 목록

- 좌측 패널에 프로젝트의 모든 테스트 런이 표시됩니다.
- 런 이름, 라운드, 버전, 상태(진행 중/완료)가 표시됩니다.
- 런을 클릭하면 우측에 결과 그리드가 표시됩니다.

### 7-3. 결과 입력

| 방법 | 설명 |
|---|---|
| 🖱️ 마우스 | 결과 셀 클릭 → 드롭다운에서 PASS/FAIL/BLOCK/N/A/NS 선택 |
| ⌨️ 단축키 | 셀 선택 후 P(Pass), F(Fail), B(Block), N(NS) 키로 빠른 입력 |
| 📋 범위 채우기 | Ctrl+D로 현재 값을 아래로 복사, Shift+Click으로 범위 선택 |
| ↩️ Undo/Redo | Ctrl+Z로 실행 취소, Ctrl+Shift+Z로 재실행 (최대 200단계) |

### 7-4. 첨부파일

- 결과 행에 이미지/문서 파일을 첨부할 수 있습니다.
- 지원 형식: 이미지(PNG, JPEG, GIF, BMP, WebP), 문서(PDF, DOC, XLSX, PPTX), ZIP, TXT, CSV, LOG
- 파일 크기 제한: **50MB**
- 이미지 파일은 미리보기가 제공됩니다.
- 드래그 앤 드롭으로 이미지를 첨부할 수도 있습니다.

### 7-5. 런 관리

| 기능 | 설명 |
|---|---|
| 완료 | 런 상태를 "완료"로 변경 |
| 복제 | 기존 런을 복제. 모든 결과가 NS로 초기화 |
| 삭제 | 런과 모든 결과, 첨부파일을 영구 삭제 |
| 내보내기 | 런 결과를 Excel 파일로 다운로드 |

💡 **TIP**: 내장 타이머 기능으로 각 TC의 수행 시간을 기록할 수 있습니다.

## 8. 비교

> 📎 `12_compare_tab.png`

두 개의 테스트 런을 선택하여 결과를 나란히 비교할 수 있습니다.

1. 좌측(기준) 런과 우측(대상) 런을 각각 선택합니다.
2. 비교 테이블에서 각 TC별 좌/우 결과를 확인합니다.
3. 필터로 **변경된 항목만** 또는 **리그레션만**(PASS→FAIL) 볼 수 있습니다.

ℹ️ **통계 정보**: 상단에 총 TC 수, 변경 건수, 리그레션 건수, 개선 건수가 표시됩니다.

## 9. 대시보드

> 📎 `13_dashboard_top.png`

선택한 테스트 런의 결과를 다양한 차트와 통계로 분석합니다.

### 9-1. 요약 카드

- **전체 TC**: 테스트 케이스 총 개수와 실행률
- **PASS**: 통과 건수 및 비율
- **FAIL**: 실패 건수 및 비율
- **BLOCK**: 블록 건수 및 비율
- **N/A**: 해당없음 건수 및 비율
- **미수행**: 미실행 건수 및 비율

### 9-2. 차트

> 📎 `14_dashboard_charts.png`

| 차트 | 설명 |
|---|---|
| 결과 분포 (도넛) | PASS/FAIL/BLOCK/N/A/미수행 비율을 도넛 차트로 시각화 |
| 라운드별 비교 (바) | 라운드(R1, R2...)별 결과 추이를 스택 바 차트로 표시 |
| 우선순위별 분포 | 우선순위별 TC 수, PASS/FAIL/BLOCK/N/A/미수행 건수 테이블 |

| 담당자별 현황 | 담당자별 할당/완료/통과율 테이블 |
| 실패 히트맵 | 카테고리 × 우선순위 매트릭스의 실패 건수를 색상 강도로 표시 |

## 10. 리포트

> 📎 `16_report_tab.png`

테스트 런 결과를 요약 리포트로 확인하고, PDF/Excel로 다운로드할 수 있습니다.

1. 리포트 탭에서 테스트 런을 선택합니다.
2. 요약 섹션: 총 TC, 결과별 건수/비율 등을 확인합니다.
3. 실패 목록: FAIL 결과의 TC 상세를 확인합니다.
4. **PDF 다운로드**: 서식화된 리포트 파일 (회의/보고용)
5. **Excel 다운로드**: 상세 데이터 파일 (추가 분석용)

| `Shift + Click` | 범위 선택 | 테스트 수행 그리드 |
| `Ctrl + C / V` | 복사 / 붙여넣기 | TC 관리 그리드 |
| `Enter` | 셀 편집 확정 | 모든 그리드 |
| `Escape` | 편집 취소 / 모달 닫기 | 전역 |
